TypeScript generics are one of the most powerful features in the language. They allow you to write components and functions that work with *any* type while maintaining complete type safety.

## Why Generics?

Without generics, you'd face a dilemma: write type-safe code that only works with one type, or use `any` and lose all safety. Generics give you the best of both worlds.

```typescript
// Without generics — loses type information
function identity(arg: any): any {
  return arg;
}

// With generics — preserves type information
function identity<T>(arg: T): T {
  return arg;
}

const result = identity("hello"); // type: string ✓
```

## Constraint Patterns

Real power comes from *constraining* your generics. The `extends` keyword lets you specify a minimum shape:

```typescript
interface HasId {
  id: string;
}

function findById<T extends HasId>(items: T[], id: string): T | undefined {
  return items.find(item => item.id === id);
}
```

This function works with **any** type that has an `id` string property. The returned type preserves the full original type — not just `HasId`.

## Conditional Types

TypeScript's conditional types let you create type-level if/else logic:

```typescript
type IsString<T> = T extends string ? true : false;

type A = IsString<"hello">;  // true
type B = IsString<42>;       // false
```

### The `infer` Keyword

The real magic happens with `infer`, which lets you *extract* types from other types:

```typescript
type ReturnType<T> = T extends (...args: any[]) => infer R ? R : never;

type Result = ReturnType<() => string>; // string
```

> Think of `infer` as pattern matching for types. It declares a type variable that TypeScript fills in for you.

## Practical Utility Types

Here are some utility types I use daily:

```typescript
// Make specific keys optional
type PartialBy<T, K extends keyof T> = Omit<T, K> & Partial<Pick<T, K>>;

// Deep readonly
type DeepReadonly<T> = {
  readonly [K in keyof T]: T[K] extends object
    ? DeepReadonly<T[K]>
    : T[K];
};

// Extract non-nullable keys
type RequiredKeys<T> = {
  [K in keyof T]-?: {} extends Pick<T, K> ? never : K;
}[keyof T];
```

## Key Takeaways

1. **Start simple** — Use basic generics before reaching for advanced patterns
2. **Constrain early** — The `extends` keyword is your friend
3. **Think in types** — Conditional types are pattern matching for the type system
4. **Use built-in utilities** — TypeScript ships with `Partial`, `Required`, `Pick`, `Omit`, and many more

Mastering generics takes time, but it fundamentally changes how you think about type safety. Your future self will thank you.
