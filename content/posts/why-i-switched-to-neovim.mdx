I used VS Code for five years. It's a phenomenal editor — great extension ecosystem, solid performance, and an excellent out-of-the-box experience. So why switch?

## The Tipping Point

It started with a simple observation: **my hands were leaving the keyboard too often**. Reaching for the mouse to select text, navigate files, or click a button in the UI. Each instance takes less than a second, but over a full day of coding, it adds up.

> The fastest code editor is the one that never makes you think about the editor.

## Modal Editing Changes Everything

The core insight of Vim (and by extension, Neovim) is **modal editing**. Instead of one mode where keys type characters, you have distinct modes:

- **Normal mode** — Navigate and manipulate text
- **Insert mode** — Type text (like a regular editor)
- **Visual mode** — Select text
- **Command mode** — Run editor commands

```
d2w    → Delete 2 words
ci"    → Change inside quotes
ggVGd  → Delete entire file
```

It feels alien at first. Then it clicks. You start *thinking* in text operations: "delete inside brackets," "change until semicolon," "yank this paragraph." Your editing vocabulary grows exponentially.

## My Setup

I built my config from scratch using **lazy.nvim** as the plugin manager:

```lua
-- Core plugins
{
  "nvim-telescope/telescope.nvim",  -- Fuzzy finder
  "nvim-treesitter/nvim-treesitter", -- Syntax highlighting
  "neovim/nvim-lspconfig",          -- LSP support
  "hrsh7th/nvim-cmp",               -- Autocompletion
  "lewis6991/gitsigns.nvim",         -- Git integration
}
```

The key principle: **only add plugins that solve a real problem**. Resist the urge to replicate VS Code feature-for-feature.

## The Learning Curve

Let's be honest — the first two weeks are *painful*. You'll be slower at everything. The temptation to switch back is real!

My advice:

1. **Use `vimtutor`** — Built-in tutorial, takes 30 minutes
2. **Learn one new motion per day** — Don't try to memorize everything at once
3. **Disable mouse support** — Force yourself to use keyboard navigation
4. **Keep a cheat sheet** — Refer to it, but resist the urge to look up every motion

## Should You Switch?

**Yes, if:**
- You spend most of your day in a text editor
- You enjoy optimizing your workflow
- You're comfortable with terminal-based tools

**No, if:**
- Your editor is not a bottleneck (honest assessment)
- You rely heavily on GUI debugging tools
- You don't want to invest 2–4 weeks in reduced productivity

The truth is, the editor matters less than people think. What matters is that your tools get out of your way. For me, Neovim does that better than anything else.
